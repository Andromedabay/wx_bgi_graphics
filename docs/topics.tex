\section*{Topics}

\begin{description}
  \item[Classic BGI API] Borland-style graphics compatibility entry points
  \item[Advanced OpenGL Extension API] Optional runtime/window/context helpers that complement classic BGI calls
  \item[User Input Hook Registration] Register optional user-supplied callbacks that fire after the library has completed its own internal event processing
  \item[Mouse Scroll API] Functions for reading accumulated scroll wheel input
  \item[Input Default-Behavior Control] API for bypassing or re-enabling the library's built-in input processing
  \item[Hook-Context DDS Functions] Functions safe to call from within user hook callbacks
  \item[PNG Export API] Save the BGI framebuffer or a DDS camera view directly to a PNG file
  \item[wxWidgets Embedding API] C API functions for hosting the BGI surface inside a wxGLCanvas
  \item[Standalone wx Window API] WxPython-style App/Frame/MainLoop API for Python, Pascal, and simple C programs
  \item[Camera Viewport API] 
  \item[2-D Camera Convenience API] A 2-D camera is an orthographic camera that looks straight down at the XY ground plane (Z = 0) in the Z-up world
  \item[User Coordinate System (UCS) API] A UCS defines a named local coordinate frame inside the Z-up, right-handed world space
  \item[World Extents API] The world extents define an axis-aligned bounding box (AABB) in world space that represents the programmer's declared drawing area
  \item[User Input Hook Callbacks] Optional user-supplied callbacks invoked after the library's own internal event processing
\end{description}
